TERMINAL

Directorio = Carpeta

Lista comandos
  • ls = List de Directorios/Carpetas
  • cd = Change Directori: Poner cd + el nombre de la carpeta. Para autocompletar el nombre de la carpeta escribir las primeras letras y darle al tabulador
  • mkdir = Make Directori
  • rmidr = remove directory: solo remueve el directorio no los archivos
  • cp = Copy Directori
  • mc = Move
  • rm = Remove
  • find = find/encontrar
  • pwd = Print Working Directori: imprime la ruta posisionado actualmente
  • - Flags = especificación de un comando previo / a=list all
  • ls * -r / ls -R: recursivo: muestra todas las carpetas y todos sus archivos
  • clear: borrar toda la linia de comando
  • control c: detener un proceso y volver a la ruta
  • * = ir al ultimo archivo que haya dentro de la carpeta
  • rm -r = eleiminar un directorio
  • ls -a = lista de los documentos ocultos
  • code . = para abrir un documento de codigo des de la terminal

GIT

Estados

  • Working Directory: el moment donde estamos trabajando y modificando archivos(git add)
  • Staging Area: es la area de espera. Es el espacio donde se van a guardar los archivos editados del Working Directory(git add) y se hace la foto (git commit)
  • Local Repository: toma la foto anterior y lo guarda en el repositorio local

Cuando se hacen git add + git commit: cuando se hace un paro del trabajo para guardar todos los cambios

Nomenclaturas de lenguajes

HTML: etiquetas

CSS: selectores/ selectores con dos palabras se aplica el kebab case: cuando hay dos palabras el espacio que hay en blanco se remplaza por un guión=> -

JS: camelCase para declarar las variables, funciones, arrays, objetos literales de JavaScript. Hay una excepción que es con las clases, en este caso se declaran con la primera letra en MAYU.

REACT: PascalCase => cuando son dos palabras compuestas las primeras letras de las palabras van con MAYUS. Ej: LoginForm

PYTHON: snake_case => cuando son dos palabras compuestas las dos van en MINUS separadas con un guión bajo _. Ej: login_form
